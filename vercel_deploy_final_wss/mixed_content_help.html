🔒 混合内容问题解决方案

⚠️ 检测到混合内容错误
您的应用通过HTTPS访问，但尝试连接到不安全的WebSocket端点 (ws://)。 浏览器的安全策略阻止了这种连接。
📋 问题说明
当网页通过HTTPS加载时，浏览器要求所有资源（包括WebSocket连接）也必须使用安全协议。 这是为了保护用户数据安全的重要安全机制。

🛠️ 解决方案

方案1: 允许不安全内容 (临时解决)

在当前页面临时允许混合内容：

Chrome
Firefox
Safari
Edge
  1. 点击地址栏右侧的 🛡️ (盾牌) 图标
  2. 选择 "加载不安全脚本" 或 "允许不安全内容"
  3. 页面会自动刷新，MQTT连接应该能正常工作
  1. 点击地址栏左侧的 🔒 (锁) 图标
  2. 点击 "连接不安全"
  3. 选择 "禁用此会话的保护"
  4. 刷新页面
  1. Safari对混合内容较为严格
  2. 建议使用其他浏览器或HTTP版本
  3. 或者在Safari设置中调整安全选项
  1. 点击地址栏右侧的 🛡️ 图标
  2. 选择 "关闭此页面的保护"
  3. 确认并刷新页面

方案2: 使用HTTP版本 (推荐测试)

在本地环境使用HTTP版本，避免混合内容限制：

  1. 在本地运行: flutter run -d chrome --web-port=8080
  2. 访问: http://localhost:8080
  3. HTTP环境下可以正常使用WS连接
打开HTTP版本

方案3: 配置WSS服务器 (永久解决)

在EMQX服务器上启用WSS支持：

  1. 为服务器配置SSL证书
  2. 启用EMQX的WSS监听器 (端口8084)
  3. 更新应用配置使用WSS连接
  4. 重新部署应用

详细步骤请参考: EMQX_WSS配置完整指南.md

方案4: 使用代理服务

通过HTTPS代理转发WebSocket连接：

  1. 部署WebSocket代理服务
  2. 代理服务提供WSS接口
  3. 转发到后端WS服务器
  4. 更新应用配置使用代理

📞 需要帮助？

  • 检查网络连接和防火墙设置
